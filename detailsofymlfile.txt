
## 1Ô∏è‚É£ `name: Simple Python Action`

```yaml
name: Simple Python Action
```

* This is the **workflow name**
* It appears in the **GitHub ‚Üí Actions tab**
* You can name it anything (e.g., `Python CI`, `Run Python Script`)

---

## 2Ô∏è‚É£ `on: [push]`

```yaml
on: [push]
```

* This tells GitHub **when to run the workflow**
* Here, it runs **every time you push code** to the repository
* Example triggers:

  * `push`
  * `pull_request`
  * `workflow_dispatch` (manual trigger)

---

## 3Ô∏è‚É£ `jobs:` section

```yaml
jobs:
  run-python:
```

* A **job** is a set of steps executed together
* `run-python` is the **job name**
* You can have multiple jobs (build, test, deploy, etc.)

---

## 4Ô∏è‚É£ `runs-on: ubuntu-latest`

```yaml
runs-on: ubuntu-latest
```

* Specifies the **virtual machine** where the job runs
* `ubuntu-latest` = Linux machine provided by GitHub
* Other options:

  * `windows-latest`
  * `macos-latest`

---

## 5Ô∏è‚É£ `steps:` section

```yaml
steps:
```

* Steps are **executed in order**
* Each step performs **one action**

---

## 6Ô∏è‚É£ Step 1: Checkout code

```yaml
- name: Checkout code
  uses: actions/checkout@v3
```

üìå **What it does:**

* Downloads your **GitHub repository code** into the runner
* Without this step, `hello.py` would **not be available**

‚úî Mandatory for almost all workflows

---

## 7Ô∏è‚É£ Step 2: Setup Python

```yaml
- name: Setup Python
  uses: actions/setup-python@v4
  with:
    python-version: '3.10'
```

üìå **What it does:**

* Installs **Python 3.10** on the runner
* Sets it as the default Python version

‚úî Ensures your code runs with the **correct Python version**

---

## 8Ô∏è‚É£ Step 3: Run Python file

```yaml
- name: Run Python file
  run: python hello.py
```

üìå **What it does:**

* Executes the command:

  ```bash
  python hello.py
  ```
* Runs your Python script inside GitHub‚Äôs server

‚úî If `hello.py` prints output, it will appear in **Actions logs**
